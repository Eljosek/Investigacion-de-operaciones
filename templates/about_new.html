¿Qué es la Programación Lineal?

Método Gráfico

Para 2 variables:

La programación lineal es una técnica de optimización matemática que busca encontrar el mejor resultado (máximo o mínimo) de una función objetivo lineal, sujeta a un conjunto de restricciones lineales.

  • Visualización gráfica de restricciones

    • Los elementos principales son:

  • Identificación de región factible
    • Evaluación de puntos críticos
    • Variables de decisión: Las incógnitas que queremos determinar (x₁, x₂, ..., xₙ)
    • Solución geométrica intuitiva
    • Función objetivo: La expresión que queremos maximizar o minimizar
  • Restricciones: Las limitaciones del problema (desigualdades o igualdades)
  • Condiciones de no-negatividad: xᵢ ≥ 0 para todas las variables
    • 2 variables máximo

La programación lineal es una técnica de optimización matemática que busca encontrar el mejor resultado (máximo o mínimo) de una función objetivo lineal, sujeta a un conjunto de restricciones lineales.

Función Objetivo

Expresión matemática que queremos optimizar:

Método Gráfico Maximizar: Z = c₁x₁ + c₂x₂ + ... + cₙxₙ

Minimizar: Z = c₁x₁ + c₂x₂ + ... + cₙxₙ

El método gráfico es una técnica visual para resolver problemas de programación lineal con dos variables.

Restricciones

Limitaciones que debe satisfacer la solución:

Graficar las restricciones
a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ ≤ b₁

Cada restricción se representa como una línea recta en el plano cartesiano.

a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ ≥ b₂
x₁, x₂, ..., xₙ ≥ 0 (no negatividad)
Identificar la región factible

Área que satisface todas las restricciones simultáneamente.

Encontrar los vértices

Puntos de intersección que forman las esquinas de la región.

Evaluar la función objetivo

El óptimo siempre está en un vértice de la región factible.

Tecnologías Implementadas
Ventaja: Permite visualizar claramente la solución y la región factible.

Limitación: Solo funciona con problemas de 2 variables.
Backend (Python)
  • Flask: Framework web ligero y flexible
  • PuLP: Biblioteca de optimización lineal
  • CBC Solver: Motor de optimización profesional
  • NumPy: Cálculos numéricos y matriciales

    • Método Simplex
    Frontend
  • Bootstrap 5: Framework CSS responsivo
  • Font Awesome: Iconografía moderna

    • El método Simplex es el algoritmo algebraico más utilizado para resolver problemas de programación lineal con cualquier número de variables.

  • Chart.js: Gráficos interactivos
  • JavaScript: Interactividad del usuario
    • Proceso del Algoritmo:
  • Forma estándar: Convertir todas las restricciones a igualdades usando variables de holgura/exceso
  • Solución básica inicial: Comenzar en un vértice de la región factible
  • Iteraciones: Moverse de vértice en vértice mejorando el valor objetivo
  • Optimalidad: Detenerse cuando no hay mejora posible
    • Garantía de Precisión: Todos los resultados están validados matemáticamente usando el solver CBC de categoría industrial.
    Variables Auxiliares:
  • Variables de holgura (s): Para restricciones ≤ (convierten en igualdad)
  • Variables de exceso (e): Para restricciones ≥ (parte que excede)
  • Variables artificiales (a): Para Fase 1 del método Two-Phase

    • Algoritmos y Características Two-Phase Simplex: Si hay restricciones ≥ o =, se usa una Fase 1 para encontrar una solución básica factible inicial, y una Fase 2 para optimizar.

    Método Gráfico

      Método Dual Simplex
    • Visualización 2D completa
    • Región factible coloreada
  • Evaluación de vértices
  • Ideal para aprendizaje

    • El Dual Simplex es una variante del método Simplex que parte de una solución dual factible pero puede tener restricciones primales violadas.

    ¿Cuándo usar Dual Simplex?
    • Re-optimización: Cuando se modifican restricciones de un problema ya resuelto
    • Análisis de sensibilidad: Estudiar cómo cambia la solución con cambios paramétricos
    • Branch and bound: En algoritmos de programación entera
      • Simplex Clásico
      Diferencias clave:
    • Tableaux paso a paso
    • Variables auxiliares automáticas
    • Historial de iteraciones
      • Simplex Primal:
    • Soporte para ≤, ≥, =
    • Parte de solución primal factible
    • Mejora valor objetivo
    • Mantiene factibilidad primal
    Dual Simplex
    Simplex Dual:
      • Parte de solución dual factible
      • Conversión primal-dual
      • Restaura factibilidad primal
      • Teoría de dualidad
      • Mantiene optimalidad dual
      • Optimización en espacio dual
    • Extracción de solución primal
    Nota: Esta implementación usa automáticamente el método más adecuado según las características del problema.

    Formato de Entrada
    Industria y Producción
      Función Objetivo:
    • Optimización de procesos de manufactura
      • Planificación de la producción
      • maximizar z = 3x1 + 2x2 o max z = 3x1 + 2x2
      • Asignación de recursos limitados
      • minimizar z = 2x1 - x2 + 3x3 o min z = 2x1 - x2 + 3x3
      • Minimización de costos operativos
    Restricciones:
    • x1 + 2x2 <= 8 (menor o igual)
      • Finanzas y Logística
    • 2x1 + x2 >= 6 (mayor o igual)
      • x1 + x2 = 5 (igualdad)
      • Optimización de carteras de inversión
      • x1 >= 0, x2 >= 0 (no-negatividad)
      • Planificación de rutas de transporte
    • Gestión de inventarios
    • Asignación de presupuestos
    Consejos:
    • Las variables deben nombrarse como x1, x2, x3, etc.
    • Las restricciones de no-negatividad son opcionales (se asumen automáticamente)
    • Usa espacios para mayor claridad
